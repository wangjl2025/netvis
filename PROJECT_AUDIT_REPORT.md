```
备份统计：
├─ backup-20260326-1016 (183KB)
├─ backup-20260326-1333 (187KB)

├─ backup-20260326-1434 (207KB)
├─ backup-20260326-1445 (218KB)
├─ backup-20260326-1503 (219KB)
├─ backup-20260326-1713 (221KB)
├─ backup-20260326-2030 (222KB)
├─ backup-20260326-2147 (226KB)
├─ backup-20260326-2200 (228KB)
├─ backup-20260326-2220 (238KB)
└─ backup-20260327-0008 (256KB)  ← 最新
```

**问题**：
- 12 个备份占用 ~2.5MB 磁盘空间
- 备份间隔不规律（有时 10 分钟，有时 1 小时）
- 无备份清理策略
- 项目根目录显得杂乱

**建议**：
- 保留最近 3 个备份（当天 + 前两天）
- 删除其余备份，释放 ~2MB 空间
- 建立备份策略：每天 EOD 自动备份一次，保留 7 天

#### 2. **检查脚本过多且无文档**
```
检查脚本列表（共 20+ 个）：
├─ arch_check.py / arch_check2.py / arch_check3.py
├─ check_align.py
├─ check_all.py
├─ check_cong2.py / check_cong_parent.py
├─ check_deep.py
├─ check_detail.py
├─ check_dns_kb.py
├─ check_dom.py
├─ check_fields.py
├─ check_issues.py
├─ check_nat_multi.py
├─ check_tcpcong.py / check_tcpcong2.py
├─ check_week2.py
├─ diag.py
├─ final_check.py

├─ verify.py / verify2.py / verify3.py / verify4.py / verify5.py / verify6.py
└─ verify_badge.py / verify_fix.py / verify_mobile.py
```

**问题**：
- 脚本命名无规律（check_* vs verify_* vs arch_check_*）
- 无 README 说明各脚本用途
- 版本号混乱（check_cong2.py vs check_tcpcong2.py）
- 无法判断哪些脚本已过时
- 占用项目根目录空间

**建议**：
- 创建 `scripts/` 目录，整理所有检查脚本
- 编写 `scripts/README.md` 说明各脚本用途
- 删除过时脚本（保留最新版本）
- 建立脚本命名规范

#### 3. **HTML 文件体积接近警戒线**
```
index.html: 256KB（警戒线 280KB）
ui-prototype.html: 226KB
test_cong.html: 256KB
test_cong_minimal.html: 13KB
test_cong_svg.html: 2.5KB
```

**问题**：
- index.html 已占 256KB，距离 280KB 警戒线仅 24KB
- 每增加一个新协议或功能，都有爆表风险
- 多个 test_*.html 文件，不清楚是否还在使用

**建议**：
- 审计 index.html 中是否有冗余 CSS/JS
- 考虑将大型 CSS 块拆分为 modules/style-*.css
- 删除或归档过时的 test_*.html 文件
- 建立体积监控机制（CI/CD 中添加体积检查）

#### 4. **数据文件命名不一致**
```
data/ 目录中：
├─ tcp3.js（占位符，实际由 rebuildStepData() 生成）
├─ tcp4.js
├─ congestion.js（但代码中引用为 tcpcong）
├─ tcpcong.js（重复？）
├─ ...
```

**问题**：
- tcp3.js 是占位符，容易误导
- congestion.js vs tcpcong.js 命名混乱
- 文件名与 protocolDB key 不一致风险

**建议**：
- 删除 tcp3.js 占位符（或改名为 tcp3.placeholder.js）
- 统一 congestion.js 和 tcpcong.js（选一个）
- 在 MEMORY.md 中明确记录"文件名必须与 protocolDB key 一致"
