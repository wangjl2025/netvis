## 🐛 发现的问题

### 🔴 **严重问题**

#### 1. **HTML 体积接近警戒线**
```
当前：257KB
警戒线：280KB
剩余空间：23KB
```

**风险**：
- 每增加一个新协议或功能，都有爆表风险
- 新增 drill.js 模块已占用部分空间

**建议**：
- 审计 index.html 中是否有冗余 CSS/JS
- 考虑将大型 CSS 块拆分为 modules/style-*.css
- 建立体积监控机制（CI/CD 中添加体积检查）

#### 2. **knowledge 数量不对齐**（多个协议）
```
DNS：4 条 knowledge vs 8 步 stepData ← 不对齐
DNS-iter：4 条 knowledge vs 8 步 stepData ← 不对齐
```

**影响**：
- 某些步骤无对应的知识点显示
- 用户学习体验不完整

**建议**：
- DNS：补充 4 条知识点
  - DNS 记录类型（A/AAAA/CNAME/MX/NS）
  - TLD 与权威 DNS 的区别
  - DoH（DNS over HTTPS）
  - DNS 缓存实践

---

### 🟡 **中等问题**

#### 3. **TCP 三次握手占位符**
```
data/tcp3.js 是占位符，实际数据由 rebuildStepData() 生成
```

**问题**：
- 容易误导开发者
- 不清楚为什么这个文件是空的

**建议**：
- 改名为 `tcp3.placeholder.js`
- 或在文件顶部添加更详细的注释

#### 4. **演练模式（drill.js）的题目来源**
```
从 protocolDB[*].quiz 自动收集题目
```

**问题**：
- 某些协议的 quiz 数量较少（3-4 道）
- 演练模式的题目库可能不够丰富

**建议**：
- 为每个协议补充更多思考题（目标 5-8 道）
- 考虑添加难度分级（简单/中等/困难）

#### 5. **多模式协议的 UI 一致性**
```
DNS 和 NAT 都有模式切换条
但 UI 样式和交互可能不一致
```

**建议**：
- 统一模式切换条的 UI 样式
- 统一交互逻辑（点击切换、动画效果）

### 🟢 **轻微问题**

#### 6. **字段 derive 使用不一致**

**建议**：
