### TCP拥塞控制三重Bug修复 (2026-03-26 22:51)
用户反馈 tcpcong 画布空白，经过深度排查发现三个独立Bug：

**Bug 1 - 根本原因（画布空白）**：CSS 里有 `#congestionDiagram { display:none; }` 规则，导致 JS 执行 `el.style.display = ''` 时，inline style 被清空后 CSS 规则生效，元素仍然隐藏。
- 修复：移除 CSS 中的 `display:none`；JS 改为对 canvas-inner 类元素用 `display='flex'`

**Bug 2（底部旁白框内容错误）**：`stepNarration`/`narrationText` 是 tcp3 专用的通用旁白框，tcpcong 切换后没有隐藏它，导致底部还显示 tcp3 的默认文字。
- 修复：switchProtocol 里 tcpcong 时隐藏 stepNarration，切换其他协议时恢复

**Bug 3（进度条旁协议名硬编码）**：进度条旁边 `<span>TCP 三次握手</span>` 是硬编码，不随协议切换更新。
- 修复：改为 `id="progressProtoName"`，在 switchProtocol 里动态更新

### 第二轮排查（2026-03-26 23:19）
用户反馈修复后还是空白。经深度排查发现两个更底层的 CSS 布局问题：

**Bug 4 - SVG height:auto 在 flex 中折叠为 0**：`.cong-chart-wrap svg` 设了 `height:auto`，但在 flex 容器里 SVG 的 height:auto 无法根据 viewBox 自动计算，导致高度=0，内容不可见。
- 修复：加 `min-height:160px`

**Bug 5 - canvas-inner 的 align-items:center 压缩子元素宽度**：`canvas-inner` CSS 有 `align-items:center`，配合 `flex-direction:column` 时，`cong-chart-wrap`（width:100%）被居中但宽度被压缩。
- 修复：`#congestionDiagram` 覆盖 `align-items:stretch; justify-content:flex-start`

### 第三轮修复（2026-03-26 23:40）- 彻底重构布局
用户反馈还是不行。放弃在 canvas-inner flex 布局上修补，改用和其他画布一致的绝对定位方案：
- **根本改变**：移除 `congestionDiagram` 上的 `canvas-inner` 类，改为 `position:absolute; inset:0` 填满父容器
- 同步更新 JS 画布显示逻辑：`canvasId === 'congestionDiagram'` 时也用 `display:flex`
- **教训**：`canvas-inner` 的 `height:100%` 在 flex 链里不可靠；绝对定位 + inset:0 才是最稳健的方案
