## 全面 Bug 修复（01:00）

### 发现并修复的 6 个 Bug
1. **[严重] tcp3 数据未写入 protocolDB** — `rebuildStepData()` 只更新了全局变量，没有写入 `protocolDB['tcp3']`，导致页面初始化时 `protocolDB['tcp3'].steps.length` 报错。修复：在函数末尾添加 `protocolDB['tcp3'] = {...}` 直接赋值。
2. **[严重] data/tcp3.js 引用主 HTML 变量** — 之前脚本生成的 tcp3.js 里用了 `stepCards`/`stepData` 等主 HTML 变量，无法独立加载。修复：改为占位符注释。
3. **[中等] PROTO_META 缺少 ssh 条目** — SSH 协议没有在 PROTO_META 注册，导致播放器标题显示 "ssh" 而非 "SSH 安全远程登录"。修复：添加了完整的 ssh 节点配置。
4. **[中等] openProtocol 懒加载守卫误触发 tcp3** — `if (!protocolDB[pid])` 对 tcp3 也会触发 loadProtocolData，但 tcp3 数据是函数生成的，不走文件加载。修复：添加 `&& pid !== 'tcp3'` 条件。
5. **[中等] switchProtocol 无懒加载守卫** — 直接点击播放器内的 tab 切换协议时，如果数据未加载会报错。修复：在 switchProtocol 开头添加懒加载守卫。
6. **[轻微] preloadAll 包含 tcp3** — 预加载列表包含 tcp3 会触发加载占位符文件。修复：从预加载列表中移除 tcp3。
